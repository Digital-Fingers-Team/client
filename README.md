## Description

A production-ready web system for managing classrooms, students, and teachers with real-time messaging, file uploads, and a React frontend.

## Project setup

### Backend Setup

```bash
# Copy environment file
cp .env.example .env

# Fill in your DATABASE_URL, UPLOADS_DIR, and JWT_SECRET in .env

# Install backend dependencies
npm install

# Generate Prisma client
npx prisma generate

# Run database migrations
npx prisma migrate dev --name init

# Seed the database with sample data
npm run seed

# Start the backend application
npm run start:dev
```

### Frontend Setup

```bash
# Navigate to client directory
cd client

# Install frontend dependencies
npm install

# Start the React development server
npm start
```

The backend will run on `http://localhost:3000` and the frontend on `http://localhost:3001`.

## Features

- User authentication with JWT
- Role-based access control (Admin, Teacher, Student)
- Course management
- Real-time messaging with WebSockets
- File upload and storage
- Dashboard with statistics and charts
- User management

## API Endpoints

### Authentication
- `POST /auth/login` - Login with email and password

### Users
- `GET /users` - List all users

### Stats
- `GET /stats/overview` - Get overview statistics (Admin only)
- `GET /stats/engagement` - Get engagement statistics (Admin only)

### Conversations
- `GET /conversations` - List user's conversations
- `POST /conversations` - Create a new conversation

### Messages
- `GET /conversations/:id/messages` - Get messages in a conversation
- `POST /conversations/:id/messages` - Send a message in a conversation

### Files
- `POST /files` - Upload a file (Admin/Teacher only)
- `POST /files/uploads` - Upload a file for messaging
- `GET /files` - List files with optional filters
- `GET /files/:id` - Get file metadata
- `GET /files/:id/download` - Download file
- `DELETE /files/:id` - Delete file (Admin/Teacher only)

## Database

This project uses Prisma ORM with SQLite for development. For production, you can switch to PostgreSQL by updating the `DATABASE_URL` in your `.env` file.

## File Uploads

Files are stored locally in the `uploads/` directory. The storage is abstracted for easy migration to cloud storage (e.g., AWS S3).
